# AppTalentNavi - サービス仕様書

## 1. サービス概要
- **一言で**: 自律型AIエージェントにビジネスタスクを丸投げする体験を提供する研修ツール
- **対象ユーザー**: ビジネスパーソン（非エンジニア含む）、研修受講者
- **解決する課題**: AIエージェントの実務活用イメージを体験的に習得。データ抽出・Web調査・ファイル整理を通じてAIの自律的なタスク遂行を理解する
- **ステータス**: MVP v2.0

## 2. ユーザーロールと権限
| ロール | できること |
|---|---|
| 研修受講者 | CLI対話でビジネスタスク依頼、データ抽出、Web調査、ファイル整理体験 |
| 研修講師 | Codespaces Secrets設定、カスタムデータ追加、シナリオカスタマイズ |

## 5. データモデル
### セッション
- JSONL形式で `~/.local/state/co-vibe/sessions/` に保存
- 会話履歴を保持し、`--resume` で復元可能

### 研修データ
- `data/meetings/` に営業議事録テキストファイル（20件）
- 4種類のフォーマット: 構造化、半構造化、自由記述、混合

### スキル定義
- `skills/` にMarkdownファイルとしてスキル定義を配置
- エージェントのシステムプロンプトから参照

## 6. 外部連携
- LLM（優先）: Gemini API（gemini-2.5-flash-lite）
- LLM（フォールバック）: Ollama（ローカル、http://localhost:11434）
- 推奨Ollamaモデル: qwen2.5-coder:7b
- クラウドIDE: GitHub Codespaces / Gitpod

## 7. ビジネスルール
- Gemini APIキーがあればクラウドAIを優先使用
- Geminiが使えない場合はOllamaにフォールバック（完全ローカル動作）
- クラウドIDE環境ではOllamaフォールバックなし（Gemini必須）
- 日本語対応必須
- 教育目的のため、各ツール実行前に説明を表示

## 8. 非機能要件
- 想定ユーザー数: 研修1回あたり10-30名
- 動作環境: Windows / macOS / Linux（Python 3.8+）
- Gemini利用時: インターネット接続必要、APIキー必要（無料枠あり）
- Ollama利用時: RAM 8GB以上推奨（qwen2.5-coder:7b）
- ゼロ依存: pip install 不要

## 9. 既知の課題・制限
- Ollama のモデル品質に依存（7Bモデルでは複雑なタスクが困難な場合あり）
- Gemini API の無料枠には利用制限あり
- WebSearch/WebFetch はインターネット接続が必要

## 11. 競合分析・ポジショニング

### v1.x → v2.0 ピボットの背景

v1.x では「CLIでLP作成」を体験価値としていたが、Claude Artifacts / v0.dev / Bolt.new が無料で即座にLP生成可能であり、**差別化要因ゼロ**と判断。v2.0で「AIエージェントの自律動作を可視化する体験」へピボットした。

### 構成要素の関係

> **Ollama** = AIエンジン（部品）
> **co-vibe** = AIエージェントフレームワーク（開発者向け汎用ツール）
> **AppTalentNavi** = 非技術者がAIエージェントの自律動作を体験できる研修プロダクト

### Ollama単体との差

| 観点 | Ollama単体 | AppTalentNavi |
|---|---|---|
| インストール | 手動DL → 手動インストール → `ollama pull` | ワンコマンドで自動 |
| できること | チャットのみ（テキスト生成） | **自律型エージェント**（ファイル操作・コマンド実行） |
| ツール実行 | なし | Bash/ファイルI/O/Glob/Grep等を自律的に使用 |
| 教育的価値 | 「AIと会話できる」だけ | 「AIが自律的にタスクを完遂する過程」を体験 |

### co-vibe単体との差

| 観点 | co-vibe単体 | AppTalentNavi |
|---|---|---|
| 対象ユーザー | 開発者・技術者 | **非技術者（高校生・ビジネスパーソン）** |
| セットアップ | APIキーの手動設定が必要 | 自動検出・自動セットアップ |
| UI/表示 | 英語、技術用語 | **日本語ラベル**、教育的な表示 |
| 体験設計 | フリーフォーム | **3つのシナリオメニュー**で誘導 |
| 同梱データ | なし | 20件の会議メモ（リアルな乱雑データ） |
| 配布形態 | Pythonスクリプト | **exe化**可能 → USBで配布 |

### 競合製品との比較

| 製品 | 自律性 | 非技術者向け | プロセス可視化 | オフライン | 研修設計 |
|---|---|---|---|---|---|
| **AppTalentNavi** | **高** | **高** | **全過程表示** | **可** | **あり** |
| Ollama単体 | 低 | 中 | なし | 可 | なし |
| co-vibe単体 | 高 | 低 | あり | 可 | なし |
| Claude Artifacts | 中 | 高 | 結果のみ | 不可 | なし |
| v0.dev / Bolt.new | 中 | 高 | 結果のみ | 不可 | なし |
| Claude Code / Cursor | 高 | 低 | あり | 不可 | なし |

### ポジショニング

**カテゴリ**: AIエージェント体験研修ツール（非エンジニア向け）

LP作成ツールとしては競合優位ゼロだったが、「エージェント自律動作の可視化体験」というカテゴリでは明確な差別化を獲得した。

### 独自価値（5つの強み）
1. **プロセスの可視化** — 結果だけでなく「AIがファイルを読む→分析→書き出す」全過程を見せる
2. **リアルなビジネスデータ** — 20件の乱雑な会議メモで現実的な体験
3. **ゼロ依存** — pip install不要、exeにすればPythonすら不要
4. **オフライン完結** — Ollama使用時はインターネット不要（企業内研修で重要）
5. **研修パッケージ** — 90分カリキュラム、講師ガイド、トラブルシューティングガイド付き

### 残存する弱点
- CLI前提のため非エンジニアには心理的ハードルがある（Codespaces/Gitpodで緩和）
- Gemini無料枠のレート制限により、研修30名同時実行は厳しい可能性がある
- Ollamaフォールバック時のモデル品質（7Bクラスの限界）
