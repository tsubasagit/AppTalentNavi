# co-vibe Improvement Proposals

Generated by improvement-scout agent. Excludes known bugs (BUG-1 through BUG-11).

---

## 1. UX Improvements

### IMP-U1: `/resume` slash command shows no session picker
**Priority:** HIGH | **Effort:** S
- `/resume` in the interactive loop is not implemented as a slash command. The user must restart with `--resume` or `--session-id`.
- Proposal: Add `/resume` as a slash command that lists recent sessions and lets the user pick one to switch to without restarting.
- Location: `main()` slash command dispatch (line ~7630+)

### IMP-U2: No `/watch` status indicator
**Priority:** LOW | **Effort:** S
- When `/watch` is ON, there is no persistent visual indicator (e.g., in the scroll region footer or prompt). The user can forget it is active.
- Proposal: Show a `[W]` badge in the prompt or scroll region status when file watcher is active.
- Location: `TUI.get_input()` (line ~6024), `main()` scroll region status update (line ~7597)

### IMP-U3: Session list display is rudimentary
**Priority:** MEDIUM | **Effort:** S
- `--list-sessions` shows raw session IDs and byte sizes. No preview of what the session was about.
- Proposal: Show the first user message (or project directory) alongside each session for quick identification. `Session.list_sessions()` already has access to the file; reading the first JSONL line is cheap.
- Location: `Session.list_sessions()` (line ~5781)

### IMP-U4: `/help` does not mention `/resume` or `/providers`
**Priority:** LOW | **Effort:** S
- `/resume` is referenced in `_slash_commands` tab-completion list but is not in the help text. `/providers` is also absent from some places.
- Location: `TUI.show_help()` (line ~6626)

### IMP-U5: No progress indication during `/commit` diff generation
**Priority:** LOW | **Effort:** S
- When `git diff --cached` is large, there is a noticeable pause with no feedback before the spinner starts for LLM generation.
- Proposal: Show a brief "Collecting diff..." message before calling git.
- Location: `/commit` handler (line ~7837)

### IMP-U6: Error messages not localized for CJK users
**Priority:** MEDIUM | **Effort:** M
- The tool already detects CJK locale and adapts hints/banners, but error messages from tools (e.g., "Error: file not found") are always in English.
- Proposal: For critical user-facing errors, add bilingual messages when `_is_cjk` is True.
- Location: Various tool `execute()` methods

### IMP-U7: Missing `--sidecar` CLI flag
**Priority:** MEDIUM | **Effort:** S
- `sidecar_model` can be set via config file but not via CLI argument. Users who want to override the compaction model per-run must edit the config.
- Proposal: Add `--sidecar` argument to `_load_cli_args()`.
- Location: `Config._load_cli_args()` (line ~857)

## 2. Performance Improvements

### IMP-P1: `ReadTool` counts remaining lines by iterating entire file
**Priority:** HIGH | **Effort:** S
- After reading the requested slice, `ReadTool.execute()` does `remaining = sum(1 for _ in f)` which reads the entire rest of the file just to show "X of Y total lines". For multi-GB files (under the 100MB limit), this is O(n) wasted I/O.
- Proposal: Use `os.popen("wc -l")` or simply omit the total count and say "more lines available" without counting them.
- Location: `ReadTool.execute()` (line ~2670)

### IMP-P2: `GrepTool` opens files twice (binary probe + content search)
**Priority:** MEDIUM | **Effort:** S
- `search_file()` opens the file once for the binary probe (8KB read) then opens it again for content search. For large directory trees with thousands of files, this doubles the syscall overhead.
- Proposal: Combine into a single open: read the first 8KB, check for nulls, then continue reading from the same handle.
- Location: `GrepTool.execute() -> search_file()` (line ~3255)

### IMP-P3: `_build_system_prompt` walks up 10 directory levels on every call
**Priority:** LOW | **Effort:** S
- The system prompt is rebuilt only once at startup, so this is not hot-path. But if it were ever called per-request, the 10-level directory walk would be wasteful. Currently fine; noting for awareness.
- Location: `_build_system_prompt()` (line ~1266)

### IMP-P4: `_estimate_tokens()` is O(n) per character for CJK detection
**Priority:** LOW | **Effort:** M
- `_estimate_tokens()` iterates every character to check CJK ranges. For very long messages (100K+), this is noticeable.
- Proposal: Use a compiled regex `re.findall(r'[\u3000-\u9fff...]', text)` and count the length, which is faster in CPython due to C-level regex engine.
- Location: `Session._estimate_tokens()` (line ~5394)

### IMP-P5: Schema cache invalidation in ToolRegistry
**Priority:** LOW | **Effort:** S
- `ToolRegistry.get_schemas()` caches schemas but `_cached_schemas` is set to `None` on every `register()`. After all tools are registered at startup, this is fine. But if tools were dynamically added (MCP servers), each registration invalidates the cache.
- Proposal: This is already fine for the current usage. Noting for future consideration.
- Location: `ToolRegistry.register()` (line ~5023)

## 3. Robustness / Edge Cases

### IMP-R1: `MultiProviderClient.chat()` infinite retry loop potential
**Priority:** HIGH | **Effort:** S
- When all providers are rate-limited and there are no healthy fallbacks, the code does `tried_providers.discard(provider)` and `continue`, which can loop indefinitely. While there is a practical limit (the `while True` will eventually exhaust all providers), there is no explicit max-retry counter for the outer loop.
- Proposal: Add a `max_total_retries` counter (e.g., 10) to the `while True` loop to guarantee termination.
- Location: `MultiProviderClient.chat()` (line ~2089)
- **Note:** This overlaps with BUG-4 but is a broader robustness concern beyond just the infinite loop.

### IMP-R2: `WriteTool` / `EditTool` race condition on temp file cleanup
**Priority:** LOW | **Effort:** S
- Both tools use `tempfile.mkstemp()` + `os.replace()`. If the process is killed between `mkstemp` and `replace`, orphaned `.vibe_tmp` files accumulate. These are harmless but messy.
- Proposal: Add a periodic cleanup of stale `.vibe_tmp` files (older than 1 hour) in the working directory.
- Location: `WriteTool.execute()` (line ~2857), `EditTool.execute()` (line ~2997)

### IMP-R3: `FileWatcher._scan()` follows no symlinks but does not handle permission errors gracefully
**Priority:** LOW | **Effort:** S
- `os.walk()` can raise `PermissionError` for directories the user cannot read. The `_poll_loop` catches all exceptions, but the scan silently misses those directories without logging.
- Proposal: Log a debug warning when directories are skipped due to permissions.
- Location: `FileWatcher._scan()` (line ~4698)

### IMP-R4: Session file corruption on concurrent access
**Priority:** MEDIUM | **Effort:** M
- If two co-vibe instances run in the same directory and both try to save the same project session, the atomic `os.replace()` means last-write-wins. The earlier session's data is lost.
- Proposal: Use a lock file (`sessions/{id}.lock`) or include PID in the session filename.
- Location: `Session.save()` (line ~5676)

### IMP-R5: `_load_dotenv()` skips symlinks for .env but does not log a warning
**Priority:** LOW | **Effort:** S
- If the user intentionally symlinks `.env` (common in deployment setups), they get no feedback that it was skipped.
- Proposal: Print a debug-level warning when a symlink .env is skipped.
- Location: `Config._load_dotenv()` (line ~746)

### IMP-R6: `_html_to_text()` is very simplistic
**Priority:** MEDIUM | **Effort:** M
- The HTML-to-text converter in `WebFetchTool` strips all tags and collapses whitespace. This loses all structure: lists become run-on text, tables are unreadable, links lose their URLs.
- Proposal: Preserve basic structure: keep `<a href>` URLs in `[text](url)` format, render `<li>` as `- item`, preserve `<pre>/<code>` blocks.
- Location: `WebFetchTool._html_to_text()` (line ~3478)

### IMP-R7: `GitCheckpoint.rollback()` does not handle stash conflicts
**Priority:** LOW | **Effort:** S
- `git stash pop` can fail with merge conflicts. The error message just says "Rollback failed" without guidance.
- Proposal: Detect conflict state and suggest `git stash show` or `git checkout --theirs/--ours`.
- Location: `GitCheckpoint.rollback()` (line ~4567)

## 4. Code Quality / Maintainability

### IMP-C1: Monolithic 8200-line single file
**Priority:** HIGH | **Effort:** L
- The entire application is a single Python file. While this simplifies distribution, it hinders maintainability, testing, and collaboration.
- Proposal: Split into a package structure:
  - `co_vibe/config.py` - Config, constants
  - `co_vibe/providers.py` - MultiProviderClient, SSE parsing
  - `co_vibe/tools/` - One file per tool class
  - `co_vibe/tui.py` - TUI, ScrollRegion, InputMonitor
  - `co_vibe/agent.py` - Agent, orchestration
  - `co_vibe/session.py` - Session management
  - `co_vibe/main.py` - Entry point, command dispatch
  - `co_vibe/__main__.py` - `python -m co_vibe` support
- Keep a single-file distribution option via a build script that concatenates.

### IMP-C2: `main()` function is 800+ lines with deeply nested command dispatch
**Priority:** HIGH | **Effort:** M
- The `main()` function handles all slash commands in a single `if/elif` chain. This is hard to extend and test.
- Proposal: Extract slash command handlers into a dict mapping or a `CommandDispatcher` class. Each command becomes a standalone function.
- Location: `main()` (line ~7425-8248)

### IMP-C3: Duplicate code in `_is_protected_path()`
**Priority:** LOW | **Effort:** S
- Line 2779: `for dirname in ("co-vibe", "co-vibe"):` iterates the same string twice. This is a copy-paste artifact.
- Location: `_is_protected_path()` (line ~2779)

### IMP-C4: `MultiProviderClient = MultiProviderClient` (line 2204)
**Priority:** LOW | **Effort:** S
- Backward compatibility alias that assigns the class to itself. This is a no-op and should be removed.
- Location: Line 2204

### IMP-C5: Inconsistent error message formatting
**Priority:** MEDIUM | **Effort:** M
- Some tools return `"Error: ..."`, others `"Error -..."`, others just exception text. The agent loop checks for both `"Error:"` and `"Error -"` prefixes (line ~7268).
- Proposal: Standardize all tool errors to use `"Error: "` prefix. Consider a `ToolError` exception class.
- Location: All tool `execute()` methods

### IMP-C6: Magic numbers scattered throughout
**Priority:** MEDIUM | **Effort:** S
- Many numeric constants are inline: `30000` (output truncation), `10000` (sub-agent output), `50000` (web content), `500000` (PDF), etc.
- Proposal: Consolidate into a `Limits` class or module-level constants with descriptive names.
- Location: Various

### IMP-C7: No type hints
**Priority:** MEDIUM | **Effort:** L
- The codebase has no type annotations. Adding them would improve IDE support, catch bugs early, and serve as documentation.
- Proposal: Incrementally add type hints, starting with public APIs (Tool.execute, Session methods, Config fields).

### IMP-C8: No unit tests
**Priority:** HIGH | **Effort:** L
- There are zero tests. Critical logic like XML tool call extraction, token estimation, message compaction, and provider fallback have no test coverage.
- Proposal: Add a `tests/` directory with:
  - `test_tools.py` - Tool execution (mock filesystem)
  - `test_xml_extraction.py` - XML tool call parsing
  - `test_session.py` - Compaction, save/load
  - `test_config.py` - Config loading, validation
  - `test_provider.py` - Provider selection, fallback logic

## 5. New Feature Proposals

### IMP-F1: Streaming cost tracker
**Priority:** HIGH | **Effort:** M
- Users have no visibility into API costs. co-vibe supports multiple providers with different pricing.
- Proposal: Track cumulative input/output tokens per provider per session. Show with `/tokens` and on exit. Optionally estimate cost using known pricing tables.
- Location: `Agent.run()` token reconciliation (line ~7096), `TUI.show_status()`

### IMP-F2: Tool output folding / expandable detail
**Priority:** MEDIUM | **Effort:** M
- Tool results show only 3 lines by default. For debugging, users often want to see more.
- Proposal: Add a `/verbose` toggle that increases `max_detail` from 3 to 20 lines. Or store recent tool outputs and add `/last` to view the full output of the last tool call.
- Location: `TUI.show_tool_result()` (line ~6410)

### IMP-F3: Configurable tool timeout per tool type
**Priority:** MEDIUM | **Effort:** S
- All Bash commands share the same timeout (default 120s). Long-running builds need more time, while simple commands should fail faster.
- Proposal: Allow per-tool timeout configuration in `.co-vibe.json`:
  ```json
  {"tool_timeouts": {"Bash": 300, "WebFetch": 30}}
  ```
- Location: `BashTool.execute()` (line ~2308)

### IMP-F4: Export conversation to Markdown
**Priority:** LOW | **Effort:** S
- Sessions are stored as JSONL, which is not human-readable. Users may want to share or review conversations.
- Proposal: Add `/export` command that writes the current session as a formatted Markdown file.
- Location: New slash command in `main()`

### IMP-F5: `.co-vibe/hooks` pre/post tool execution hooks
**Priority:** LOW | **Effort:** M
- Power users want to run custom scripts before/after certain tools (e.g., auto-format after Write, backup before Edit).
- Proposal: Support hook scripts in `.co-vibe/hooks/pre-Write.sh`, `.co-vibe/hooks/post-Edit.sh`, etc. Execute with a short timeout.
- Location: Agent tool execution loop (line ~7252)

### IMP-F6: Smart context injection from git diff
**Priority:** MEDIUM | **Effort:** M
- When the user asks the agent to work on code, the agent often needs to read files to understand context. If there are uncommitted changes (git diff), injecting a summary automatically saves round-trips.
- Proposal: On first user message, if in a git repo with uncommitted changes, auto-inject a condensed `git diff --stat` into the system prompt.
- Location: `_build_system_prompt()` (line ~1124)

### IMP-F7: Ollama / local model support
**Priority:** HIGH | **Effort:** M
- The tool only supports cloud providers (Anthropic, OpenAI, Groq). Many users want to use local models via Ollama.
- Proposal: Add Ollama as a provider. Ollama exposes an OpenAI-compatible API at `http://localhost:11434/v1`. The `_chat_openai_compat()` method already handles OpenAI-compatible APIs, so this is mostly configuration:
  - Detect `OLLAMA_HOST` env var or `CO_VIBE_OLLAMA_URL`
  - Add models like `ollama:llama3.1:8b` to the model registry
  - Skip SSRF check for localhost
- Location: `MultiProviderClient.__init__()` (line ~1350)

## 6. Security Improvements

### IMP-S1: API keys briefly visible in debug output
**Priority:** HIGH | **Effort:** S
- When `--debug` is enabled, HTTP request headers are not logged, but the API key is used in `_chat_anthropic()` and `_chat_openai_compat()`. If a user copies debug output, the key prefix `sk-...` appears in error messages from the API (line ~1744: `err_msg = error_body[:300]`).
- Proposal: Redact API key prefixes (`sk-`, `key-`, etc.) from error messages before printing.
- Location: `MultiProviderClient._http_request()` (line ~1727)

### IMP-S2: `BashTool._build_clean_env()` allowlist could miss new sensitive vars
**Priority:** MEDIUM | **Effort:** S
- The environment sanitizer uses a fixed allowlist + blocklist. New sensitive environment variables (e.g., from new cloud services) might slip through.
- Proposal: Invert the logic: default-deny all environment variables except those in `_ALWAYS_ALLOW`, plus explicitly allowlisted prefixes (e.g., `LANG_`, `LC_`, `XDG_`).
- Location: `BashTool._build_clean_env()` (line ~2266)

### IMP-S3: WebSearchTool User-Agent leaks version info
**Priority:** LOW | **Effort:** S
- The User-Agent string `co-vibe/1.4.0 (+https://github.com/ochyai/co-vibe)` reveals the exact version. This is fine for transparency but could help attackers target known vulnerabilities in specific versions.
- Proposal: Consider making the version in User-Agent optional (configurable via env var).
- Location: `WebSearchTool._ddg_search()` (line ~3552), `WebFetchTool.execute()` (line ~3439)

### IMP-S4: No rate limiting on SubAgent/ParallelAgents tool calls
**Priority:** MEDIUM | **Effort:** S
- A malicious prompt could cause the agent to spawn many SubAgents in a loop, each making API calls. While `SubAgentTool.HARD_MAX_TURNS = 20` caps individual agents, there is no limit on how many SubAgents the main agent can spawn per session.
- Proposal: Add a session-level counter for SubAgent/ParallelAgents invocations (e.g., max 20 per session).
- Location: `SubAgentTool.execute()` (line ~4119), `ParallelAgentTool.execute()` (line ~4962)

### IMP-S5: MCP server processes not killed on crash
**Priority:** MEDIUM | **Effort:** S
- If co-vibe crashes (e.g., unhandled exception, SIGKILL), the `atexit` handler may not run, leaving MCP server subprocesses orphaned.
- Proposal: Set MCP server processes as children of the main process group, so they are killed on parent exit. On Unix, use `prctl(PR_SET_PDEATHSIG)` or rely on process group cleanup.
- Location: `MCPClient.start()` (line ~4328)
