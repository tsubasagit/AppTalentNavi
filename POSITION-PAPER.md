# AppTalentNavi v2.0 — 製品ポジショニング分析

*2026-03-02*

---

## 概要

AppTalentNaviは、非技術者（高校生・ビジネスパーソン）がAIエージェントの自律動作を体験できる研修プロダクトである。本ドキュメントでは、構成要素であるOllama・co-vibeとの関係を明確化し、AppTalentNaviの独自価値を定義する。

> **Ollama** = AIエンジン（部品）
> **co-vibe** = AIエージェントフレームワーク（開発者向け汎用ツール）
> **AppTalentNavi** = 非技術者がAIエージェントの自律動作を体験できる研修プロダクト

---

## 1. Ollama単体との比較

| 観点 | Ollama単体 | AppTalentNavi |
|---|---|---|
| **インストール** | 手動DL → 手動インストール → `ollama pull` | ワンコマンドで自動（DL→インストール→モデル取得） |
| **モデル管理** | ユーザーが自分で選んでpull | 推奨モデル（qwen2.5-coder:7b）を自動選択・取得 |
| **できること** | チャットのみ（対話型Q&A） | **自律型エージェント**（ファイル操作・コマンド実行・Web作成を自動遂行） |
| **ツール実行** | なし（テキスト生成のみ） | Bash/ファイルI/O/Glob/Grep等をAIが自律的に使う |
| **教育的価値** | 「AIと会話できる」だけ | 「AIが自律的にタスクを完遂する過程」を目の前で体験 |

**最大の差**: Ollamaは「AIモデルの実行環境」に過ぎない。AppTalentNaviは、そのAIに「手足（ツール）」を与え、実際のビジネスタスクを自律遂行させる。

## 2. co-vibe単体との比較

| 観点 | co-vibe単体 | AppTalentNavi (hajime.py + co-vibe) |
|---|---|---|
| **対象ユーザー** | 開発者・技術者 | **非技術者（高校生・ビジネスパーソン）** |
| **セットアップ** | APIキーの手動設定、プロバイダー選択が必要 | 自動検出・自動セットアップ（Gemini→Ollama自動フォールバック） |
| **UI/表示** | 英語、技術用語（Bash, Glob等） | **日本語ラベル**、教育的な表示（「シェル (Bash)」等） |
| **エラーメッセージ** | 英語、traceback表示 | 日本語、技術情報を隠してユーザーフレンドリーに |
| **体験設計** | フリーフォーム（何でもできるが何をすべきか不明） | **3つのシナリオメニュー**で体験を誘導 |
| **同梱データ** | なし | 20件の会議メモ（リアルな乱雑データ） |
| **スキル定義** | なし | `skills/*.md` でシナリオ別のAI行動ガイド |
| **配布形態** | Pythonスクリプト | **exe化**（PyInstaller）可能 → USBで配布 |
| **Cloud IDE対応** | 手動設定 | Codespaces/Gitpod自動検出 |

**最大の差**: co-vibeは「万能AIエージェントエンジン」。AppTalentNaviは、それを **「1000人の高校生が迷わず使える研修ツール」** にパッケージングしたもの。

## 3. AppTalentNaviの独自価値（他にない強み）

### 3.1 プロセスの可視化

Claude Artifacts / v0.dev は結果だけ見せる。AppTalentNaviは「AIがファイルを読む → 分析する → 書き出す」の**全過程**を見せる。

### 3.2 リアルなビジネスデータ

20件の乱雑な会議メモという「現実的な」データで体験。構造化・半構造化・自由記述・メール転送風の混在フォーマット。

### 3.3 ゼロ依存

pip install不要、Pure Python。exeにすればPythonすら不要。

### 3.4 オフライン完結

Ollama使用時はインターネット不要（企業内研修で重要）。

### 3.5 研修パッケージ

90分カリキュラム、講師ガイド、トラブルシューティングガイド付き。

## 4. アーキテクチャ

```
┌─────────────────────────────────────────────┐
│  AppTalentNavi (hajime.py)                  │
│  ├─ 環境検出（Cloud IDE / ローカル）          │
│  ├─ LLM自動セットアップ（Gemini or Ollama）   │
│  ├─ 体験メニュー（3シナリオ）                 │
│  └─ 作業ディレクトリ選択                      │
├─────────────────────────────────────────────┤
│  co-vibe.py (AIエージェントエンジン)          │
│  ├─ HAJIME_MODE: 日本語UI / エラー抑制       │
│  ├─ スキル読み込み (skills/*.md)             │
│  └─ ツール自律実行 (Bash/File/Glob/Grep...)  │
├─────────────────────────────────────────────┤
│  ollama_setup.py (LLM環境管理)              │
│  ├─ 自動インストール (Windows)               │
│  ├─ サービス起動管理                          │
│  └─ モデル自動取得 + 進捗表示                 │
├─────────────────────────────────────────────┤
│  LLM Provider                               │
│  ├─ Gemini 2.5 Flash Lite (クラウド)         │
│  └─ Ollama + qwen2.5-coder:7b (ローカル)    │
└─────────────────────────────────────────────┘
```

### レイヤー説明

| レイヤー | 責務 | ファイル |
|---|---|---|
| **ランチャー** | 環境検出、LLM自動セットアップ、体験メニュー表示 | `hajime.py` |
| **エージェントエンジン** | ツール自律実行、HAJIME_MODE（日本語UI/エラー抑制）、スキル読み込み | `co-vibe.py` |
| **LLM環境管理** | Ollamaインストール・起動・モデル取得の自動化 | `ollama_setup.py` |
| **LLMプロバイダー** | Gemini API（クラウド優先）/ Ollama（ローカルフォールバック） | 外部サービス |

### データフロー

```
ユーザー入力
    │
    ▼
hajime.py: 環境検出 → Gemini or Ollama 自動選択
    │
    ▼
co-vibe.py: HAJIME_MODE で起動
    │
    ├─ skills/*.md をシステムプロンプトに注入
    ├─ ユーザーの指示を LLM に送信
    ├─ LLM がツール呼び出しを決定
    │   ├─ Glob: ファイル探索
    │   ├─ Read: ファイル読み取り
    │   ├─ Write: ファイル書き出し
    │   └─ Bash: コマンド実行
    └─ 結果を日本語ラベル付きで表示
```

## 5. 競合ポジショニングマップ

### 横軸: 自律性、縦軸: 非技術者への親和性

```
    高い親和性
      │
      │  ★ AppTalentNavi
      │     (自律エージェント × 研修パッケージ)
      │
      │            Artifacts / v0.dev
      │            (結果表示のみ × 簡単)
      │
──────┼──────────────────────────────── 高い自律性
      │
      │  Ollama          co-vibe
      │  (チャットのみ)   (フル機能 × 開発者向け)
      │
      │            Claude Code / Cursor
      │            (高機能 × エンジニア専用)
      │
    低い親和性
```

### 競合製品との差別化サマリー

| 製品 | 自律性 | 非技術者向け | プロセス可視化 | オフライン | 研修設計 |
|---|---|---|---|---|---|
| **AppTalentNavi** | **高** | **高** | **全過程表示** | **可** | **あり** |
| Ollama単体 | 低（チャットのみ） | 中 | なし | 可 | なし |
| co-vibe単体 | 高 | 低 | あり | 可 | なし |
| Claude Artifacts | 中（サンドボックス） | 高 | 結果のみ | 不可 | なし |
| v0.dev / Bolt.new | 中（Web生成特化） | 高 | 結果のみ | 不可 | なし |
| Claude Code / Cursor | 高 | 低（エンジニア専用） | あり | 不可 | なし |

## 6. ターゲットユーザーと利用シーン

### プライマリターゲット

- **高校生・大学生**: AIエージェント体験授業（90分完結）
- **新入社員研修**: AI活用リテラシー研修
- **ビジネスパーソン**: DXリスキリング講座

### 利用シーン

| シーン | 規模 | 推奨構成 | ネットワーク |
|---|---|---|---|
| 学校の授業 | 30-40名 | Codespaces + Gemini API | 必要 |
| 企業研修（社内） | 10-20名 | ローカルPC + Ollama | 不要 |
| セミナー・展示 | 1-5名 | exe配布 + Ollama | 不要 |
| オンライン研修 | 制限なし | Codespaces + Gemini API | 必要 |

## 付録: co-vibe（upstream）について

- **Repository**: [github.com/ochyai/co-vibe](https://github.com/ochyai/co-vibe)
